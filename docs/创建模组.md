# 如何创建 StarRailExpress 扩展模组

本文档面向希望为 StarRailExpress 编写独立扩展模组的开发者，涵盖从环境准备到项目构建的完整流程。  
关于代码层面的 API（注册角色、技能、事件等），请参阅 **[api.md](api.md)**。

---

## 重要提醒
CCA组件尽量少同步，否则会造成网络拥挤。

对于倒计时，建议客户端模拟计算，或者存储触发时间。实在需要手动倒计时，建议服务端仅在**10s同步一次**，客户端同步计算。

## 创建新项目

### 方式一：使用 Fabric 模板生成器（推荐）

访问 [Fabric 模板生成器](https://fabricmc.net/develop/template/) 填写以下信息后下载模板：

- **Minecraft Version：** `1.21.1`
- **Project Name / Mod ID：** 填写你的模组名称与 ID
- **Package Name：** 填写你的 Java 包名

下载解压后进入项目目录，继续配置构建文件（见下节）。

### 方式二：手动建立目录结构

```
你的模组/
├── src/
│   └── main/
│       ├── java/
│       │   └── com/example/yourmod/
│       │       └── YourMod.java
│       └── resources/
│           ├── fabric.mod.json
│           └── yourmod.mixins.json   # 如需 Mixin
├── gradle/
│   └── wrapper/
│       ├── gradle-wrapper.jar
│       └── gradle-wrapper.properties
├── build.gradle
├── gradle.properties
├── gradlew
├── gradlew.bat
└── settings.gradle
```

## 配置构建文件

### gradle.properties

```properties
# Gradle 选项
org.gradle.caching=true
org.gradle.parallel=true
org.gradle.jvmargs=-Xmx2G

# Minecraft / Fabric 版本（需与 StarRailExpress 保持一致）
minecraft_version=1.21.1
loader_version=0.17.2
fabric_version=0.116.7+1.21.1

# 你的模组属性
mod_version=1.0.0
maven_group=com.example
archives_base_name=your_mod_id
```

### build.gradle

```groovy
plugins {
    id 'fabric-loom' version '1.14-SNAPSHOT'
    id 'maven-publish'
}

version = project.mod_version
group = project.maven_group

base {
    archivesName = project.archives_base_name
}

repositories {
    // Fabric
    maven { url "https://maven.fabricmc.net/" }
    // Cardinal Components（如需 CCA）
    maven {
        name = "Ladysnake Mods"
        url = 'https://maven.ladysnake.org/releases'
    }
    // Modrinth（Wathe 等前置）
    maven {
        name = "Modrinth"
        url = "https://api.modrinth.com/maven"
        content { includeGroup "maven.modrinth" }
    }
}

dependencies {
    minecraft "com.mojang:minecraft:${project.minecraft_version}"
    mappings loom.officialMojangMappings()
    modImplementation "net.fabricmc:fabric-loader:${project.loader_version}"
    modImplementation "net.fabricmc.fabric-api:fabric-api:${project.fabric_version}"

    // ——— StarRailExpress 依赖 ———
    // 先在本地构建 StarRailExpress（./gradlew build），
    // 再将输出的 JAR 放到本项目的 libs/ 目录中引用：
    modImplementation files("libs/star_rail_express-4.0.0.jar")

    // StarRailExpress 的前置模组（同样需要放入 libs/ 或通过 Maven 引入）
    modImplementation "maven.modrinth:wathe:1.3.2-1.21.1"
}

processResources {
    inputs.property "version", project.version
    inputs.property "minecraft_version", project.minecraft_version
    inputs.property "loader_version", project.loader_version
    filteringCharset "UTF-8"

    filesMatching("fabric.mod.json") {
        expand "version": project.version,
               "minecraft_version": project.minecraft_version,
               "loader_version": project.loader_version
    }
}

def targetJavaVersion = 21
tasks.withType(JavaCompile).configureEach {
    it.options.encoding = "UTF-8"
    if (targetJavaVersion >= 10 || JavaVersion.current().isJava10Compatible()) {
        it.options.release.set(targetJavaVersion)
    }
}

java {
    def javaVersion = JavaVersion.toVersion(targetJavaVersion)
    if (JavaVersion.current() < javaVersion) {
        toolchain.languageVersion = JavaLanguageVersion.of(targetJavaVersion)
    }
    withSourcesJar()
}
```

> **如何获取 StarRailExpress JAR：**
> 1. 克隆 StarRailExpress 源码仓库
> 2. 在仓库根目录执行 `./gradlew build`（需要网络访问各 Maven 仓库）
> 3. 构建产物位于 `build/libs/star_rail_express-<version>.jar`（不带 `-sources` 后缀的那个）
> 4. 将该 JAR 复制到你的模组项目的 `libs/` 目录

### settings.gradle

```groovy
pluginManagement {
    repositories {
        maven {
            name = 'Fabric'
            url = 'https://maven.fabricmc.net/'
        }
        gradlePluginPortal()
    }
}
```

## 模组基本结构

### fabric.mod.json

```json
{
  "schemaVersion": 1,
  "id": "your_mod_id",
  "version": "${version}",
  "name": "Your Mod Name",
  "description": "你的模组描述",
  "authors": ["你的名字"],
  "license": "GPL-3.0-only",
  "environment": "*",
  "entrypoints": {
    "main": ["com.example.yourmod.YourMod"],
    "client": ["com.example.yourmod.client.YourModClient"]
  },
  "mixins": [
    "yourmod.mixins.json"
  ],
  "depends": {
    "minecraft": "1.21.1",
    "fabric-loader": ">=0.17.2",
    "starrailexpress": "*",
    "wathe": "*"
  }
}
```

### 主类示例

```java
package com.example.yourmod;

import net.fabricmc.api.ModInitializer;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class YourMod implements ModInitializer {

    public static final String MOD_ID = "your_mod_id";
    public static final Logger LOGGER = LoggerFactory.getLogger(MOD_ID);

    @Override
    public void onInitialize() {
        LOGGER.info("YourMod 初始化完成");
        // 在此注册角色、物品、事件监听器等
        // 详见 api.md
    }
}
```

## 项目构建命令

以下命令均在项目根目录下执行（Windows 使用 `gradlew.bat`，Linux/macOS 使用 `./gradlew`）：

| 命令 | 说明 |
|------|------|
| `./gradlew build` | 编译并打包模组，产物输出至 `build/libs/` |
| `./gradlew runClient` | 启动测试用 Minecraft 客户端（含你的模组） |
| `./gradlew runServer` | 启动测试用 Minecraft 服务端 |
| `./gradlew genSources` | 生成 Minecraft 反混淆源码（IDE 调试用） |
| `./gradlew clean` | 清理构建缓存 |
| `./gradlew --refresh-dependencies build` | 强制重新下载依赖后构建 |

> **构建产物说明：**
> - `build/libs/your_mod_id-<version>.jar` — 正式发布 JAR
> - `build/libs/your_mod_id-<version>-sources.jar` — 源码 JAR（可选）

## 注册扩展内容

项目搭建完成后，请参阅以下文档了解如何在代码中注册各类扩展内容：

- **[api.md](api.md)** — 完整 API 参考（角色、修饰符、技能、事件、HUD、CCA 组件等）
- **[../CreateExtention.md](../CreateExtention.md)** — 快速参考：如何注册角色、物品、实体、商店、网络包、GUI 及 Mixin

> **重要提醒：**
> - **不要引用 Wathe 的库**，它会导致崩溃（未初始化状态）。
> - 添加 Mixin 后需在 `yourmod.mixins.json` 和 `fabric.mod.json` 中同步注册。
> - 网络包遵循命名惯例：C2S 包放在 `packet/` 下，继承 `CustomPacketPayload`。
